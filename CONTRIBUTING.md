# Contributing to Functorial Narrative Analysis

Thank you for your interest in contributing to this research project!

## How to Contribute

### Reporting Issues
- Use GitHub Issues to report bugs, suggest features, or ask questions
- Check existing issues before creating a new one
- Provide as much detail as possible

### Code Contributions
1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Run tests (`make test`)
5. Run linting (`make lint`)
6. Commit with clear messages
7. Push and create a Pull Request

### Research Contributions
We welcome contributions in several areas:

- **Corpus Collection**: Help collect narratives from underrepresented traditions
- **Functor Implementation**: Implement new observation functors
- **Language Support**: Add support for additional languages
- **Validation**: Help validate results against expert annotations
- **Theory**: Contribute to the category-theoretic framework

### Academic Collaboration
If you're interested in academic collaboration:
- Open an issue with the `collaboration` label
- Describe your research interests and potential contribution
- We're especially interested in partnerships with scholars of non-Western narrative traditions

## Development Setup

```bash
# Clone the repository
git clone https://github.com/ibrahimcesar/functorial-narrative-analysis.git
cd functorial-narrative-analysis

# Set up environment
make setup

# Run tests
make test

# Run linting
make lint
```

## Code Style
- Follow PEP 8 for Python code
- Use type hints where practical
- Write docstrings for public functions
- Keep functions focused and testable

## Questions?
Open an issue or reach out via the repository discussions.
